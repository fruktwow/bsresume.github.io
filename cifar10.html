CIFAR10 - CNN

Google Colab
March 29, 2021

The CIFAR-10 dataset (Canadian Institute For Advanced Research) is a collection of images that are commonly used to train machine learning and computer vision algorithms. It is one of the most widely used datasets for machine learning research. The CIFAR-10 dataset contains 60,000 32x32 color images in 10 different classes. The 10 different classes represent airplanes, cars, birds, cats, deer, dogs, frogs, horses, ships, and trucks. There are 6,000 images of each class.

Here the goal is to build CIFAR-10 image classifier and improve the accuracy of 10 classes.

INPUT IMAGE : For input data we are using CIFAR10 dataset.

CONVOLUTIONAL LAYER : Convolutional layer is our first layer through which input data passes

Syntax : Conv2d(Input channel, Output channel, Kernel size)

Optionally we can give padding and stride parameters too.

Deep neural networks, more concretely convolutional neural networks (CNN), are basically a stack of layers which are defined by the action of a number of filters on the input. Those filters are usually called kernels. For example, the kernels in the convolutional layer, are the convolutional filters. Actually no convolution is performed, but a cross-correlation. The kernel size here refers to the widthxheight of the filter mask.

Kernel

POOLING LAYER : Pooling layer used here is Max pooling layer. The max pooling layer returns the pixel with maximum value from a set of pixels within a mask (kernel).

Pooling layer

Experiment 1

Accuracy 54%

This is a basic default one where we are using 3 input channel and 16 output channel within two convolutional layers and one pooling layer between the two convolutional layers. Kernel size used is 5x5. Matrix size is 16x5x5. This took almost 2 minutes to run with the accuracy of 54%.


Experiment 2

Accuracy 42%

Here I've added one more convolutional layers. Here 3 input channel and 16 output channel are used within 3 convolutional layers along with one pooling layer after second convolutional layer. Kernel size used is 3x3. Matrix size is 16x3x3. This took almost 3 and a half minutes to run with the accuracy of 42%. Thus adding an extra convolutional layer with same input and output channel decreases the accuracy.

Experiment 3

Accuracy 58%

After comparing previous 2 Experiments it is shown that increasing the convolutional layer with same input and output channel has adverse affect on the accuracy of the model, thus here instead I removed one convolutional layer and increased the output channel to 24 with kernel size of 5x5 and matrix of size 16x5x5. This took almost 2 and a half minutes with the accuracy of 58%.


Experiment 4

Accuracy 65%

After comparing the above 3 Experiments it is shown that increasing the output layer has increased the accuracy of the model. Thus in this experiment I increased the output channel to 732 with the kernel size of 4x4 and matrix of size 732x5x5. This experiment took almost an hour and finished with the accuracy of 65%


Experiment 5

Accuracy 69%

Here in my final experiment I tried adding an extra convolutional layer along with high output channel but instead of putting pooling layer between 2nd and 3rd convolutional layers it is now between 1st and 2nd convolutional layers with 900 output channel. Kernel size of 4x4 and matrix of size 900x2x2. This Experiment took almost 3 hours to finish with the accuracy of 69%.
